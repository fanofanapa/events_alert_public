# Gate Events Alert

Небольшой пет-проект, который следит за активностями на Gate.com и отправляет новые акции в Telegram.

Я делал его как автоматизацию для канала: чтобы не проверять руками акции, не читать каждый раз длинные условия и не копировать всё вручную в пост.

Бот забирает список активных кампаний, находит новые, вытаскивает страницу акции, чистит лишний мусор, просит LLM сделать короткое описание и отправляет результат в Telegram.

## Воркфлоу

У нас тут простой пайплайн:

1. забирает активности из API;
2. проходит по нескольким `type_id`;
3. сохраняет текущий список активностей в `activities.json`;
4. сверяет ID с уже отправленными в `known_ids.json`;
5. для новых активностей открывает страницу кампании;
6. чистит текст страницы от навигации, футеров и прочего шума;
7. отправляет очищенный текст в LLM;
8. получает короткое summary;
9. публикует пост в Telegram.

ID сохраняется только после успешной отправки. Если сообщение не ушло, активность не потеряется и бот попробует отправить её снова при следующем запуске цикла. Конечно, не полноценная retry логика, везде есть над чем работать.

## В чем смысл проекта

На биржах постоянно появляются временные акции: бонусы за депозит, торговые задания, конкурсы, airdrop-кампании и похожие штуки.

Проблема в том, что руками это неудобно:

- нужно регулярно заходить на сайт;
- нужно проверять несколько категорий;
- условия часто размазаны по длинной странице;
- часть текста на странице вообще не относится к акции;
- для Telegram всё равно нужно писать короткое человеческое описание.

Этот бот закрывает именно эту рутину. Пока что он не принимает решений стоит ли акция внимания, а просто быстро собирает и подготавливает материал для публикации. 

## Какие навыки позволил освоить этот проект

Проект простой, но в нём есть много прикладных вещей:

- работа с API через `requests`;
- cookies и headers для запросов;
- пагинация;
- хранение локального состояния;
- защита от дублей;
- Telegram Bot API;
- интеграция с LLM API;
- промптинг под конкретный формат ответа;
- чистка грязного текста со страниц;
- long-running скрипт с обработкой ошибок;


## Структура

```text
.
├── main.py
├── parser.py
├── config.py
├── requirements.txt
├── llm_summary
│   ├── config.py
│   ├── html_to_instruction.py
│   └── qwen3_coder_next.py
├── activities.json
├── known_ids.json
└── cookies.txt
```

### `main.py`

Главный файл. Запускает цикл, вызывает парсер, проверяет новые активности, формирует сообщение и отправляет его в Telegram.

### `parser.py`

Работает с Gate.com API. Забирает активности по категориям, проходит пагинацию и приводит данные к нормальному виду.

### `config.py`

Основные настройки проекта: Telegram, Gate.com endpoint, headers, задержки, список категорий.

### `llm_summary/html_to_instruction.py`

Загружает страницу кампании через Jina Reader и чистит текст. Тут отсекается большая часть мусора: футеры, навигация, картинки без смысла, языковые переключатели и похожие блоки.

### `llm_summary/qwen3_coder_next.py`

Отправляет очищенный текст в LLM и получает короткое описание акции для Telegram. Надо будет его переименовать, но это все-таки MVP, потом руки дойдут.

### `activities.json`

Последний сохранённый снимок всех активностей. Нужен больше для отладки и ручной проверки.

### `known_ids.json`

Список ID, которые уже были отправлены в Telegram.

### `cookies.txt`

Cookies для Gate.com в формате Netscape просто куки для повышения стабильности, не нужно быть зарегестрированным пользователем биржи. Файл нужен локально, куки можно достать из расширения в браузере. На самом деле не особо нужны, в планах убрать. 

## Как работает поток данных

```text
Gate.com API
    ↓
parser.py
    ↓
список активностей
    ↓
проверка новых ID
    ↓
страница конкретной акции
    ↓
очистка текста
    ↓
LLM summary
    ↓
Telegram
```

Важная часть логики - берется не заголовок акции. Открывается страница кампании и достаются условия: суммы, дедлайны, задания, ограничения. Очистка через jina reader.

После этого LLM получает уже очищенный текст, а не весь HTML страницы.

## Установка

Нужен Python 3.10+.

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Для Windows:

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## Переменные окружения

Переименовать или создать `.env` в корне проекта:

```env
TELEGRAM_BOT_TOKEN=your_telegram_bot_token
TELEGRAM_CHAT_ID=your_telegram_chat_id

CLOUD_API=your_llm_api_key
LLM_BASE_URL=your_llm_base_url
MODEL=Qwen/Qwen3-Coder-Next или другая

MAX_TOKENS=1000
TEMPERATURE=0.5
TOP_P=0.45
```

## Cookies

В корне проекта должен быть файл:

```text
cookies.txt
```

## Запуск

```bash
python main.py
```

После запуска бот будет работать в цикле:

- загрузит активности;
- сохранит текущий снимок;
- найдёт новые ID;
- подготовит summary;
- отправит новые акции в Telegram;
- подождёт заданный интервал;
- повторит всё снова.

Интервал задаётся в `config.py`:

```python
POLLING_INTERVAL = 300
```

## Настройка категорий

Категории задаются через `TYPE_IDS`:

```python
TYPE_IDS = [1, 4, 12, 14, 17, 213, 1066]
```

Это ID из URL страниц Gate.com activity center. Если нужно следить за другими категориями, можно добавить их в список.
